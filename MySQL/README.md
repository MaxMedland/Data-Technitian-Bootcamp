# 📊 SQL Data Analysis Project — Data Technician Bootcamp

This repository contains my SQL project completed during Week 3 of the **Data Technician Bootcamp**, where I worked with pre‑built relational databases to practise querying, analysing, and extracting insights from structured data. The project focused on using SQL to answer real business questions based on retail, sales, and global demographic datasets.

---

## 🚀 Project Overview

Throughout the workbook, I used the **Northwind** and **World** sample databases to complete a series of SQL tasks. These exercises covered:

- Retrieving and filtering data  
- Sorting and grouping results  
- Joining tables to combine related information  
- Understanding database design and normalisation  

The project strengthened my ability to work with relational databases and write SQL queries that support real‑world analytical needs.

---

## 🧠 Key Skills Demonstrated

### 🔍 Core SQL Querying
I used a wide range of SQL commands to explore and analyse data:

- **SELECT** — retrieving specific fields or full tables  
- **WHERE** — filtering rows based on conditions  
- **ORDER BY** — sorting results  
- **DISTINCT** — removing duplicates  
- **BETWEEN / IN** — filtering by ranges and lists  
- **LIKE** — pattern matching for text searches  

These were applied to tasks such as identifying high‑value products, filtering customers by region, and analysing order history.

---

### 🔗 Table JOINs for Multi‑Table Analysis
I worked with several JOIN types to combine data across related tables:

- **INNER JOIN**  
- **LEFT JOIN**  
- **RIGHT JOIN**  
- **FULL JOIN** (conceptual understanding)  
- **CROSS JOIN**  

These JOINs were used to:

- Link products with suppliers  
- Combine orders with customer and employee information  
- Match categories with their products  
- Analyse sales volumes across multiple tables  

---

### 📊 Aggregation & Grouping
I used **GROUP BY** and aggregate functions to summarise data:

- `COUNT()` — number of products per category  
- `SUM()` — total quantities ordered  
- `AVG()` — average population by country  

These techniques supported deeper insights into sales, inventory, and global demographics.

## ⚠️ Important Note — Database Setup Required

The SQL workbook queries **depend on two pre‑existing databases**:

- **Northwind**  
- **World**  

These databases must be **downloaded, installed, and loaded into MySQL Workbench** before running the queries.

I did **not** create these databases — I used the provided setup files.  
Without them, many queries will fail because the required tables and relationships will not exist.
